> 一个关于vector的移动构造函数的注意点

今天遇到一个有意思的问题，对于标准库里的vector容器，如果通过`push_back`、`insert`、`reserve`和`resize`函数导致vector内存重分配或者容器内部元素位置发生移动时，为了保证性能我们希望这些发生变化的元素可以“移动”到给它分配的新内存区域中，所以会用到移动构造函数。但是请看下面的代码：

```C++
#include <iostream>
#include <vector>
using namespace std;

class MoveTest {
public:
    MoveTest(int x_) : x(x_) { cout<<x<<": MoveTest()"<<endl; }
    ~MoveTest() { /*cout<<"~MoveTest()"<<endl;*/ }

    MoveTest(const MoveTest& other) {
        x = other.x;
        cout<<x<<": MoveTest(const MoveTest&)"<<endl;
    }

    MoveTest(MoveTest&& rhs) {
        x = rhs.x;
        cout<<x<<": MoveTest(MoveTest&&)"<<endl;
    }
private:
    int x;
};

int main() {
    vector<MoveTest> v;
    v.reserve(2);
    //MoveTest t1(1), t2(2), t3(3);
    v.push_back(MoveTest(1));
    v.push_back(MoveTest(2));
    v.push_back(MoveTest(3));
}
```

输出结果为：

```
1: MoveTest()
1: MoveTest(MoveTest&&)
2: MoveTest()
2: MoveTest(MoveTest&&)
3: MoveTest()
3: MoveTest(MoveTest&&)
2: MoveTest(const MoveTest&)
1: MoveTest(const MoveTest&)
```

可以看出当vector的容量为2时，当t3进入后，vector的容量会变大(不一定是2倍变成4)，之前的t1和t2却调用了拷贝构造函数。

这是因为：vector 通常保证强异常安全性，如果元素类型没有提供一个**保证不抛异常的移动构造函 数**，vector 通常会使用拷贝构造函数。而为什么会选用拷贝构造函数？这又是因为一旦某个操作发生了异常，被移动的元素已经被破坏，处于只能析构的状态，异常安全性就不能得到保证了。

因此，对于拷贝代价较高的自定义元素类型，我们应当定义移动构造函数，并标其为 noexcept。代码更改如下：

```C++
class MoveTest {
public:
    MoveTest(int x_) : x(x_) { cout<<x<<": MoveTest()"<<endl; }
    ~MoveTest() { /*cout<<"~MoveTest()"<<endl;*/ }

    MoveTest(const MoveTest& other) {
        x = other.x;
        cout<<x<<": MoveTest(const MoveTest&)"<<endl;
    }

    MoveTest(MoveTest&& rhs) noexcept {
        x = rhs.x;
        cout<<x<<": MoveTest(MoveTest&&)"<<endl;
    }
private:
    int x;
};
```

此时输出的结果为：

```
1: MoveTest()
1: MoveTest(MoveTest&&)
2: MoveTest()
2: MoveTest(MoveTest&&)
3: MoveTest()
3: MoveTest(MoveTest&&)
2: MoveTest(MoveTest&&)
1: MoveTest(MoveTest&&)
```

最后的t1和t2都使用了移动构造函数，符合我们的猜想。
