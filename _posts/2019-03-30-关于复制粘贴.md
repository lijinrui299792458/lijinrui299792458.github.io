> 复制粘贴的一些知识

“复制粘贴”是一项非常有用的发明，不论是在日常生活、写代码还是写论文上，都极大提高了生产力。这个功能是由Evelyn Bereezin发明的，试想如果没有这个功能，我们需要同一份文件的两份副本，可能就需要重复输入两次了(类似古代的抄书？)。然而Evelyn于2018年12月8日去世了，在这里缅怀一下这位"复制粘贴"伟大发明的计算机科学工作者。

## 复制粘贴功能如何实现的?

操作系统中有一个剪贴板(clipboard)，不同系统的处理细节可能不一样，但是大致的思想都类似。复制文本时会保留其样式(如果系统支持的话)，如字号、字体、颜色等等信息。所以要理解的是剪贴板里存储的不仅仅是文字或图片，而是一种带有样式的数据信息。然后粘贴时，如果应用程序同样支持该样式就会成功"复制"了。举一个形象一点的例子，我们复制一段文本或图片的时候，假设它以html的形式存储在剪贴板里，此时剪贴板里也就有了足够的信息。当粘贴时，如果复制对象是不支持html语法的应用程序，那么就会去掉一些样式或者出现格式混乱的情况。

还有一种比较特殊的处理情况，就是复制文件。不知道大家试没试过这样的操作：复制一个文件，然后删除源文件，然后粘贴时出现找不到文件的情况。按复制粘贴的原理来说，我复制了一份存储在我的剪贴板里，那么现在就应该有两份资料了，删除原来的应该不会影响剪贴板里的文件呀？ 其实不然，在复制文件时，系统只是把文件的路径复制到剪贴板，然后通过更改路径属性实现文件的克隆。至于开辟额外存储空间则是与操作系统的文件处理有关的了。

## 复制粘贴的版权保护

大家对复制粘贴现在应该有基本的了解了，然而现在是一个知识付费的时代，自己的文字都是自己的产出。所以就涉及到种种所谓的保护,如CSDN上的复制就会自己带上下面这一段:

```
作者：XXXXXXXXX
来源：CSDN 
原文：https://blog.csdn.net/XXXXXXXXXXXX
版权声明：本文为博主原创文章，转载请附上博文链接！
```

知乎上也有:

```
作者：匿名用户 
链接：https://www.zhihu.com/question/xxxxx
来源：知乎 著作权归作者所有。商业转载请联系作者获得授权，非商业转载请注明出处。
```

现在很多网站都有这样的措施，虽然并没有什么用，总有神通广大的网友会提供脚本或者方法破解限制；比如说从源代码获取文字、脚本来获取文章。但是我还是想分析一下网站做的保护措施，是如何实现当复制的内容超过一定字数时候就自动加一段文字、禁止复制这些功能的呢？

我们从头开始分析一下这个文件:

首先`140 < window.getSelection().getRangeAt(0).toString().length`是指如果csdn上复制的字数超过了140个字就会执行添加版权信息的脚本。而其中的getSelection方法是对用户选择的文本范围进行操作，返回一个Selection对象表示用户选择的文本范围或光标的当前范围。

然后为了响应浏览器的copy事件，使用了`t.preventDefault()`阻止默认的复制事件(如果用原来的复制，我们怎么加版权信息?)

在之后通过`Window.getSelection()`获取到当前选中内容，然后创建一个`<textarea>`标签，为它添加样式，即位置信息和格式信息，然后将上一步中的内容，放到文本框内。并以字符串的形式添加上额外的内容。（注：为了能在添加的内容达到换行的效果，需要使用`<textarea>`标签而不是`<input>`，需要换行处添加`'\n'`字符串）

再用``document.execCommand``将此选中的内容进行复制。其中的`document.execCommand("copy")`是html5中实现浏览器端的复制功能，语法是这样的:

```javascript
bool = document.execCommand(aCommandName, aShowDefaultUI, aValueArgument)
```

使用简单的功能是需要第一个参数`aCommandName`就ok了,其中的copy是拷贝**当前选中内容**到剪贴板。启用这个功能的条件因浏览器不同而不同，而且不同时期，其启用条件也不尽相同。(还有cut和paste：前者剪贴**当前选中的文字**并复制到剪贴板。启用这个功能的条件因浏览器不同而不同，而且不同时期，其启用条件也不尽相同。使用之前请检查浏览器兼容表，以确定是否可用。后者在光标位置粘贴剪贴板的内容，如果有被选中的内容，会被替换。剪贴板功能必须在 user.js 配置文件中启用。)

总结一下，就是监听了元素的copy事件，然后拦截了原来的复制操作，对复制过后的数据添加信息后再返回给剪贴板的api。

(至于为什么copy事件所用的js文件那么难找，我猜想可能是因为做了压缩混淆，所以可读性变的很差。)
