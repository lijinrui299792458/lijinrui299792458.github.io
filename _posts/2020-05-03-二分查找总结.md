> 二分查找总结

二分查找是一个非常基础的算法，原理是利用区间内元素是有序的，通过不断让可行区间减半，使得最终可行区间长度减为1得到答案。但是与其将其看做是一个查找问题，我觉得用边界问题来描述它更为妥当。二分查找的时候，如何界定边界是一个需要斟酌的问题，否则可能随时TLE。确保不发生TLE其实就是要满足：

- 解一直在可行区间里（区间为**左右都闭**）
- 每次判断后可行区间都会缩小

二分查找的复杂度为$O(logN)$

首先写一个我们平常写的最熟悉的二分查找：

```C++
int BinargSearc(vector<int> nums, int target) {
	int left = 0, right = nums.size() - 1;
    while (left <= right) {
        int mid = (left + right) >> 1;
        if (nums[mid] == target) return mid;
        if (nums[mid] > target) right = mid - 1;
        else left = mid + 1;
    }
    return -1;
}
```

针对这个算法模板，有这样的问题：

1.为什么是`while(left <= right)`? 

2.left和right为什么更新的时候取mid+1和mid-1呢？

回答第一个问题：因为我们这里取的是左闭右闭区间，所以当仅剩1个元素的时候仍需要再判断一次该元素是否满足，此时是`left = right`，在进入循环判断一次之后就是`left = right + 1`，此时已经不为可行区间了：[right+1, right]。所以循环条件应该为`left<=right`

回答第二个问题：试想一下，如果查找[1,2]中的2，nums[mid]取1，left在更新的时候如果是取mid而不是mid+1，那么nums[mid]的值始终为1，会造成死循环。而在这标准的二分查找模板中，是将搜索区间分成了[left,mid-1]、[mid]和[mid+1, right]三部分，判断结果的部分是在[mid]部分，如果结果不在[mid]部分再根据不等关系确定是在[left,mid-1]和[mid+1,right]中哪个部分。

然而在算法题或者面试题中不可能出这“模板”的题目，一般会有各种变式。为了方便讨论，我们不妨先做一个假设，即在下面所有的变式问题中，**数组中的元素都是满足非降序的**。

### 求左边界

这类问题一般是求数组中满足x大于等于target或大于target的第一个x的位置

由于要求满足条件的第一个x的位置，所以搜索区间为 [left, right) 左闭右开，当 nums[mid] 被检测之后，下一步的搜索区间应该去掉 mid 分割成两个区间，即 [left, mid) 或 [mid + 1, right)。。即使我们找到了`nums[mid] = target`，但是因为无法确保mid左边是否仍然有满足条件的元素，仍要向左搜索，所以要收缩右边界。一般算法模板是这样的：

```C++
int search(vector<int>& nums, int target) {
    int left = 0;
    int right = nums.size();
    while (left < right) {
        int mid = left + (right - left) / 2;
        if (nums[mid] == target) {
            right = mid;
        } else if (nums[mid] < target) {
            left = mid + 1;
        } else if (nums[mid] > target) {
            right = mid;
        }
    }
    if (left == nums.length) return -1;
	return nums[left] == target ? left : -1;
}
```

同样回答两个问题：

1.为什么是`while(left<right)`

2.为什么left和right更新值为mid+1 和 mid？

回答第一个问题，这里的循环终止条件是left<right，因为在left与right相邻时，mid取left，在循环终止时left,mid和right将位于同一个位置；如果终止条件为left<=right，那么要求终止条件应该为left = right + 1，但是在left与right相等的时候如果进入的是`right=mid`这个条件，那么整个循环会卡在left=mid=right这里。所以循环终止条件应该为left<right

回答第二个问题，如果我们找到了一个`nums[mid] = target`后，因为此mid有可能就是结果，所以下一个搜索区间应该是包含这个mid的左区间，就是[left, mid]。如果`nums[mid] < target`，则此mid不在可行区间内，所以应该在[mid+1, right]里进行搜索。如果`nums[mid] > target`，则此mid不在可行区间内，应该在[left, mid - 1]区间内继续进行搜索。

但是为了方便记忆，在这里我们不妨写一个和基础算法模板相类似的算法模板：

```C++
int search(vector<int>& nums, int target) {
    int left = 0, right = nums.size() - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2;
        if (nums[mid] < target) {
            left = mid + 1;
        } else if (nums[mid] > target) {
            right = mid - 1;
        } else if (nums[mid] == target) {
            right = mid - 1;
        }
    }
    if (left >= nums.size() || nums[left] != target) return -1;
    return left;
}
```

在该模板中区间就是左闭右闭的[left, mid-1]和[mid+1, right]了，最后判断一下是否出界就行了。

### 求右边界

这类问题一般是求数组中x小于等于target或小于target的最大x的位置

和求左边界对称着理解就可以了，搜索区间变为[left,mid)和[mid + 1,right)，算法模板如下：

```C++
int search(vector<int>& nums, int target) {
    int left = 0;
    int right = nums.size() - 1;
    while (left < right) {
        int mid = left + (right - left) / 2;
        if (nums[mid] == target) {
            left = mid + 1;
        } else if (nums[mid] > target) {
            right = mid;
        } else if (nums[mid] < target) {
            left = mid + 1;
        }
    }
    if (left == 0) return -1;
	return nums[left-1] == target ? (left-1) : -1;
}
```

同样给出方便记忆的模板：

```C++
int search(vector<int>& nums, int target) {
    int left = 0, right = nums.size() - 1;
    while (left <= right) {
        int mid = left + (right - left) / 2;
        if (nums[mid] == target) {
            left = mid + 1;
        } else if (nums[mid] > target) {
            right = mid - 1;
        } else if (nums[mid] < target) {
            left = mid + 1;
        }
    }
    if (right < 0 || nums[right] != target) return -1;
    return right;
}
```
