> C++里优先队列的自定比较方式

优先队列是一种可以从尾部插入元素，然后从头部取出优先级最高的一种数据结构。C++里的优先队列一般会用[STL库里的`priority_queue`](https://en.cppreference.com/w/cpp/container/priority_queue)。

`priority_queue`在STL内部定义的原型为：

```C++
template< class T, class Sequence=vector<T>, classCompare=less<typename Sequence::value_type> >
```

三个参数分别代表：类型、容器和比较器。第三个参数默认为less，关于less，一般通用的less结构体如下：

```C++
template <class T> 
struct less : binary_function<T, T, bool> {
 	bool operator()(const T& x, const T& y) const {
        return x < y;
    }   
}
```

通过`std::priority_queue<Typename> q`生成的默认就是大根堆，大根堆就是每次pop取出来的都是优先级值最大的元素。举个例子：往q里丢了3，6，5，2，7；经过内部排序调整，大根堆第一次返回是7，第二次是6....

`priority_queue`这种less却返回最大值的这个特性，理解的人会觉得不存在问题，但是有疑惑的人会觉得很容易理解反。比如`priority_queue`和sort都用了less结构进行比较，但是两者的优先级顺序却不同？这个在后面会讲。

优先队列里面没有对应的迭代其实现，所以只有queue顶端的元素才能够被调用。和less相反的就是greater。当然也可以自定比较方式。

首先先写个框架：

```C++
#include <iostream>
#include <queue>
using namespace std;

struct Node {
    int x;
    Node(int _x) : x(_x){}
};

int main() {
    priority_queue<Node> q;
    for (int i = 1; i <= 5; ++i) q.push(Node(i));
    while (!q.empty()) {
        cout<<q.top().x<<" ";
        q.pop();
    }
    cout<<endl;
}
```

此代码暂时无法运行，因为它少了priority_queue的核心：比较函数，程序无法知道Node这个struct如何进行比较。

## 重载运算符

```C++
#include <iostream>
#include <queue>
using namespace std;

struct Node {
    int x;
    Node(int _x) : x(_x){}
    inline bool operator<(const struct Node& other) const {
        return x < other.x;
    }
};

int main() {
    priority_queue<Node> q;
    for (int i = 1; i <= 5; ++i) q.push(Node(i));
    while (!q.empty()) {
        cout<<q.top().x<<" ";
        q.pop();
    }
    cout<<endl;
}
```

输出结果为5 4 3 2 1，我们只需要在重载运算符函数那里将小于 改成 大于 就可以输出1 2 3 4 5了。

## 构造比较函数

```C++
#include <iostream>
#include <queue>
#include <vector>
using namespace std;

struct Node {
    int x;
    Node(int _x) : x(_x){}
};

struct cmp{
    bool operator() (const Node& a, const Node &b) {
        return a.x < b.x;
    }
};

int main() {
    priority_queue<Node, vector<Node>, cmp> q;
    for (int i = 1; i <= 5; ++i) q.push(Node(i));
    while (!q.empty()) {
        cout<<q.top().x<<" ";
        q.pop();
    }
    cout<<endl;
}
```

cmp函数中如果第一个元素小于第二个元素，将内部顺序按从小到大排列，则是大顶堆。改成大于号，则表示第一个元素大于第二个元素，将内部顺序按从大到小排列，则是小顶堆。

在这里我们回答一下之前的那个问题，为什么和sort的返回优先级别有区别，首先看一下sort和`priority_queue`的两种函数原型：

```C++
template <class T, class Container = vector<T>, class Compare = less<typename Container::value_type> > 
class priority_queue;template <class RandomAccessIterator, class Compare>

template <class T, class Compare = less<typename Container::value_type> >
void sort (RandomAccessIterator first, RandomAccessIterator last, Compare comp);
```

这里的比较函数除了形式不一样外，传入其中cmp的参数顺序其实也有不同，我们在这里写个对比：

```C++
#include <iostream>
#include <queue>
#include <vector>
using namespace std;

struct Node {
    int x;
    Node(int _x) : x(_x){}
};

struct cmp{
    bool operator() (const Node& a, const Node &b) {
        cout<<"pq:"<<a.x<<"-->"<<b.x<<endl;
        return a.x < b.x;
    }
};

bool cmp2(const Node& a, const Node& b) {
    cout<<"sort:"<<a.x<<"-->"<<b.x<<endl;
    return a.x < b.x;
}

int main() {
    priority_queue<Node, vector<Node>, cmp> q;
    for (int i = 1; i <= 5; ++i) q.push(Node(i));
   /* while (!q.empty()) {
        cout<<q.top().x<<" ";
        q.pop();
    }
    cout<<endl;*/

    vector<Node> v;
    for (int i = 1; i <= 5; ++i) v.push_back(Node(i));
    sort(v.begin(), v.end(), cmp2);
  /*  for (auto it : v) cout<<it.x<<" ";
    cout<<endl;*/
}
```

输出结果如下：

```
pq:1-->2
pq:2-->3
pq:1-->4
pq:3-->4
pq:3-->5
pq:4-->5
sort:2-->1
sort:3-->2
sort:4-->3
sort:5-->4
```

当1、2传入cmp函数时，`priority_queue`中是1在前 2在后，所以cmp返回的结果会是true，即未在队列中的元素会作为第二个参数与队列中根节点进行比较；但在`sort`的cmp函数中，2在前 1在后，所以cmp返回的结果会是false。即使cmp函数的行为是一致的，但是在`priority_queue`和`sort`中会有不同的顺序。

## 题外话

最近做了一个实现LRU的算法题[https://leetcode-cn.com/problems/lru-cache/](https://leetcode-cn.com/problems/lru-cache/)，自己觉得用`priority_queue`来实现就应该可以，但是代码有bug，自己也没找出来错。在这里提到这个，是因为在编写代码时发现，如果使用了struct cmp等构造比较函数，取class Node中的某一个成员变量进行比较。若在过程中对其进行了修改，则需要等到下一次push或pop的时候才会对优先队列中的值进行重排，衣蛾比较好的方法就是放入一个无关联的值，然后再把它pop出去，这样就能及时更新了。

题目代码有些测试用例没有过，如果你有想法的欢迎私聊我：

```C++
const int N = 10010;
class LRUCache {
public:
    struct Cache {
        bool used = false;
        int value;
        int dl = -1;
    }C[N];
    int len, cnt = 0;
    struct cmp {
        bool operator() (const Cache* a, const Cache* b) {
            return (*a).dl > (*b).dl;
        }
    };
    priority_queue<Cache*, vector<Cache*>, cmp> q;
    LRUCache(int capacity) {
        len = capacity;
    }
    
    int get(int key) {
        if (C[key].used == false) return -1;
        C[key].dl = cnt++;
        q.push(&C[0]);
        q.pop();
        return C[key].value;
    }
    
    void put(int key, int value) {
        if (C[key].used) {
            C[key].value = value;
            C[key].dl = cnt++;
        } else {
            if (q.size() == len) {
                Cache* k = q.top();
                //cout<<"debug:"<<(*k).value<<" "<<(*k).dl<<endl;
                (*k).used = false;
                q.pop();
            }   
            C[key].used = true;
            C[key].value = value;
            C[key].dl = cnt++;
            q.push(&C[key]);
        }
        q.push(&C[0]);
        q.pop();
    }
};
```
