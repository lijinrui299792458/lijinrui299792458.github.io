> "只出现一次”题目

最近做题的时候遇到了"只出现一次"题目，用位运算来解决的做法比较具有代表性，在这里总结一下；当然题目可能有其他解法，但是只关注于位运算的解答上。

## 136 只出现一次的数字I

给定一个非空整数数组，除了某个元素只出现一次以外，其余每个元素均出现两次。找出那个只出现了一次的元素。

说明：

你的算法应该具有线性时间复杂度。 你可以不使用额外空间来实现吗？

示例 1:

输入: [2,2,1]
输出: 1
示例 2:

输入: [4,1,2,1,2]
输出: 4

解答：

因为只有一个元素出现1次，所以将数组中所有数异或一遍后得出的一定是剩下的元素，举个例子：2，2，3， 2 与2异或后为0，0与3异或后为3.

```C++
class Solution {
public:
    int singleNumber(vector<int>& nums) {
        int res = 0;
        for (int i = 0; i < nums.size(); ++i) res ^= nums[i];
        return res;
    }
};
```

## 137 只出现一次的数字II

给定一个非空整数数组，除了某个元素只出现一次以外，其余每个元素均出现了三次。找出那个只出现了一次的元素。

说明：

你的算法应该具有线性时间复杂度。 你可以不使用额外空间来实现吗？

示例 1:

输入: [2,2,3,2]
输出: 3
示例 2:

输入: [0,1,0,1,0,1,99]
输出: 99

解答：

可以采取逐位（32位）判断的方法，由于数组中数字只可能出现1次或3次，若某位出现次数 mod 3 后不为0，那么该位就是只出现1次的那个数上的该位。 

```C++
class Solution {
public:
    int singleNumber(vector<int>& nums) {
        int res = 0;
        for (int i = 0; i < 32; ++i) {
            int cnt = 0;
            for (auto x : nums) {
                if (x & 1 << i) cnt ++;
            }
            res |= (cnt%3)<<i;
        }
        return res;
    }
};
```

## 645 错误的集合

集合 S 包含从1到 n 的整数。不幸的是，因为数据错误，导致集合里面某一个元素复制了成了集合里面的另外一个元素的值，导致集合丢失了一个整数并且有一个元素重复。

给定一个数组 nums 代表了集合 S 发生错误后的结果。你的任务是首先寻找到重复出现的整数，再找到丢失的整数，将它们以数组的形式返回。

示例 1:

输入: nums = [1,2,2,4]
输出: [2,3]
注意:

给定数组的长度范围是 [2, 10000]。
给定的数组是无序的。

解答：

leetcode中国上的官方位运算解答里有些没有说清楚，让我们在这里根据题意举一个例子：原数组为1,2,3,4,5,6，但是误写成1,2,4,4,5,6了。所以我们可以将这12个数一起异或起来，2个相同的数字异或后就为0，这样后得到的结果就是7 = `(3^4)`，是未出现数字 和 重复出现数字 的异或。但是我们这时候其实并不知道是3和4，所以全部异或的方法不可取。

我们需要将3和4分到两个分组里，而这时我们之前求出的3^4=011^100=111可以帮助我们进行分组。我们只需要求出从右至左第一位不同的位进行分组就可以了，举个例子1,2,4,4,5,6和1,2,3,4,5,6 ，我们知道3和4异或后不同的位是最后一位，所以直接根据最后一位进行判断，相同的数字最后一位肯定相同，不同的数字会因此而分开：末位为0的数为2,2,4,4,4,6,6，末位为1的数位1,1,3,5,5。这样在2^2^4^4^4^6^6=4,1^1^3^5^5=3，就求出来3和4了。

比较重要的是如何求出从右至左第一位不同的位？答案是用lowbit：`x&-x`后的结果。在之前我们已经算出3和4的异或和为`111`，则`0111&1001`的结果为0001，所以倒数第一位是不同的，可以根据倒数第一位进行分组！

```C++
class Solution {
public:
    vector<int> findErrorNums(vector<int>& nums) {
        vector<int> res;
        int k = 0;
        for (int i = 0; i < nums.size(); ++i) {
            k ^= nums[i]^(i+1);
        }
        int lastBit = k&(-k);
        int a = 0, b = 0;
        for (int i = 0; i < nums.size(); ++i) {
            if (nums[i] & lastBit) a ^= nums[i];
            else b ^= nums[i];
            if ((i+1) & lastBit) a ^= (i+1);
            else b ^= (i+1);
        }
        auto it = find(nums.begin(), nums.end(), a);
        if (it == nums.end()) swap(a, b);
        res.push_back(a);
        res.push_back(b);
        return res; 
    }
};
```



## 260 只出现一次的数字III

一个整型数组 nums 里除两个数字之外，其他数字都出现了两次。请写程序找出这两个只出现一次的数字。要求时间复杂度是O(n)，空间复杂度是O(1)。

示例 1：

输入：nums = [4,1,4,6]
输出：[1,6] 或 [6,1]
示例 2：

输入：nums = [1,2,10,4,1,4,3,3]
输出：[2,10] 或 [10,2]

解答：

除了两个数字出现一次，其他都出现了两次，让我们找到这个两个数。思路和上面一题很类似，我们需要将这两个数分到两组里才能分别得出这两个数是什么。

通过对所有数字的一次异或操作，可以得到两个出现一次的数字的异或结果`x^y`。而这个结果其实是能帮助我们得到分组依据的，如1111 和 1001，它们异或的结果是0110，最右边出现不同的是在倒数第二位。所以可以根据倒数第二位不同将所有数字进行分组，操作是类似上面一题的。

```C++
class Solution {
public:
    vector<int> singleNumbers(vector<int>& nums) {
        int k = 0;
        for (int i = 0; i < nums.size(); ++i) {
            k ^= nums[i];
        }
        int lastBit = k & (-k);
        int a = 0, b = 0;
        for (int i = 0; i < nums.size(); ++i) {
            if (nums[i] & lastBit) a ^= nums[i];
            else b ^= nums[i];
        }
        vector<int> res;
        res.push_back(a);
        res.push_back(b);
        return res;
    }
};
```



## 287 寻找重复的数

给定一个包含 n + 1 个整数的数组 nums，其数字都在 1 到 n 之间（包括 1 和 n），可知至少存在一个重复的整数。假设只有一个重复的整数，找出这个重复的数。

```
示例 1:
输入: [1,3,4,2,2]
输出: 2
示例 2:
输入: [3,1,3,4,2]
输出: 3说明：
```

不能更改原数组（假设数组是只读的）。
只能使用额外的 O(1) 的空间。
时间复杂度小于 O(n2) 。
数组中只有一个重复的数字，但它可能不止重复出现一次。

解答：

这道题是因为确定了数字一定出现在1-n，但是出现了n+1个数字，根据鸽笼原理可知至少有一个数字发生了重复。同样出现重复的字眼，但是**可能不止出现一次**，这是一道知道出现重复却不知道重复多少次的问题。也可以从位运算的角度去思考：

比如1，2，3，4，5 和 1，2，2，3，5，可以先把每一位写出来：

| -     | 1    | 2    | 3    | 4    | 5    |
| ----- | ---- | ---- | ---- | ---- | ---- |
| 第0位 | 1    | 0    | 1    | 0    | 1    |
| 第1位 | 0    | 1    | 1    | 0    | 0    |
| 第2位 | 0    | 0    | 0    | 1    | 1    |

第0位共有3个1，第1位共有2个1，第2位共有2个1

| -     | 1    | 2    | 3    | 5    | 2    |
| ----- | ---- | ---- | ---- | ---- | ---- |
| 第0位 | 1    | 0    | 1    | 1    | 0    |
| 第1位 | 0    | 1    | 1    | 0    | 1    |
| 第2位 | 0    | 0    | 0    | 1    | 0    |

第0位共有3个1，第1位共有3个1，第2位共有1个1

可以看出1，2，3，5，2仅在第1位比1，2，3，4，5上的1的数目要多，所以按位还原后应该是$(010)_{2}$，即重复的数字应该为2。

证明其正确性：由于重复其实就意味着，使用target替换1，2，3，4，5中的任一个或几个。如果target的某一位为1，那么替换后数组中所有元素的该位的1的个数应该会保持不变或者增加1；如果target的某一位为0，那么替换后数组中所有元素的该位的1的个数应该胡保持不变或者减少1，而我们想求出target只需要考虑target中为1的位数，也就是相比原数组该位增加的位数。

```C++
class Solution {
public:
    int findDuplicate(vector<int>& nums) {
        int n = nums.size() - 1;
        int bitmax = 31;
        while (!(n >> bitmax)) bitmax--;  //求n的最高位

        int ans = 0;
        for (int bit = 0; bit <= bitmax; ++bit) {
            int x = 0, y = 0;
            for (int i = 1; i <= n + 1; ++i) {
                if (nums[i-1] & 1<<bit) x++;
                if (i <= n && i & 1<<bit) y++; 
            }
            if (x > y) ans |= 1<<bit;
        }
        return ans;
    }
};
```
