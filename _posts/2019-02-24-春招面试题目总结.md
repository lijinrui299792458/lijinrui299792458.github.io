## Python修饰器实现

装饰器可以在不修改目标函数代码的前提下, 在目标函数执行前后增加一些额外功能。关于装饰器的完整实现以及原理可以参考[这篇博客](https://www.cnblogs.com/reklen/p/9534941.html)

```python
def check_result(func):
    def wrapper(*args, **kwargs):
        '''this is wrapper'''
        result = func(*args, **kwargs)

        if result < 0:
            return 0
        else:
            return result
    return wrapper


@check_result
def foo(x, y):
    return x * y
```


## Python中`_new_`与`__init__`魔法函数的区别

首先看一下这段代码:

```python
class  User:
    def __new__(cls, *args, **kwargs):
        print("new")

    def __init__(self,name):
        self.name=name
        print("init")

user=User()
```

输出结果为:new

说明我们没有执行init函数，原因是:new的功能是在生成对象之前所做的动作，接受的参数为cls类(cls作为第一个参数用来表示类本身，self作为第一个参数用来标识类的实例)；init是在对象生成之后完善对象的属性，接受的参数为self对象。具体的过程实际上是这样的：new执行动作，return一个对象，init接受该对象并完善对对象的初始化。我们这样子写就清楚了:

```python
class  User:
    def __new__(cls, *args, **kwargs):
        print("new")
        return super().__new__(cls)

    def __init__(self,name):
        self.name=name
        print("init")

print(type(User("body")))

#output:
#new
#init
#<class '__main__.User'>
```

## 常见排序算法

### 插入排序

像卡牌游戏一样，将新的牌从与左边的牌进行比较，直到找到合适的位置进行插入。

```c++
template<class Elem>
void insertSort(Elem A[],int n)
{
    for(int i = 1;i < n;i++)
        for(int j = i;j >= 1 && A[j] < A[j-1];j--)
            swap(A,j,j-1);
}
```

时间复杂度为O(n^2)

我们也可以将数组往后移动，增加一个哨兵:

```C++
void insertSort(vector<int>& A)
{
	for (int i = 1; i < A.size(); ++i)
	{
		int tmp = A[i];
		int j;
		for (j = i; j > 0&&A[j]<A[j-1]; j--)
		{
            A[j] = A[j - 1];
		}
		A[j] = tmp;
	}
}
```

### 冒泡排序

从数组的底部一直比较到顶部，相邻元素两两比较并交换，使最小元素像"气泡"一样被推到数组顶部。

```C++
void bubSort(vector<int>& A)
{
	for (int i = 0; i < A.size()-1; ++i)
	{
		for (int j =A.size()-1; j > i; --j)
		{
			if (A[j] < A[j - 1])
				swap(A[j], A[j - 1]);
		}
	}
}
```

时间复杂度为O(n^2)

考虑优化可以增加一个变量标识一次循环中是否发生了变换，如果没发生交换说明已经有序，可以提前结束。

```C++
void bubSort(vector<int>& A)
{
	for (int i = 0; i < A.size()-1; ++i)
	{
		int flag = 1;
		for (int j =A.size()-1; j > i; --j)
		{
			if (!flag)
				break;
			if (A[j] < A[j - 1] )
			{
				swap(A[j], A[j - 1]);
				flag = 0;
			}		
		}
	}
}
```

### 选择排序

与插入排序类似，但是是找到数组中第i小的元素放到数组的第i个位置。即每次从未排序的序列中找到最小元素，并放到未排序数组的前面。

```C++
void selectSort(vector<int>& A)
{
	for (int i = 0; i < A.size(); ++i)
	{
		int index=i;
		for (int j = i; j < A.size(); ++j)
		{
			if (A[j] < A[index])
				index = j;
		}
		swap(A[i], A[index]);
	}
}
```

时间复杂度为O(n^2)

### shell排序

shell排序在不相邻的元素之间比较和交换。利用了插入排序的最佳时间代价特性，它试图将待排序序列变成基本有序的，然后再用插入排序来完成排序工作

在执行每一次循环时，Shell排序把序列分为互不相连的子序列，并使各个子序列中的元素在整个数组中的间距相同，每个子序列用**插入排序**进行排序。每次循环增量是前一次循环的1/2，子序列元素是前一次循环的2倍

最后一轮将是一次“正常的”插入排序（即对包含所有元素的序列进行插入排序）

考虑优化的话，当选择“增量每次除以3”递减时，Shell排序的平均运行时间是O(n^(1.5))

```C++
void shellSort(vector<int>& A)
{
	const int D = 3;
	for (int d = A.size() / D; d > 0; d /= D)
	{
		for (int i = 0; i < d; i++)
		{
			for (int j = i + d; j < A.size(); j += d)
			{
				for (int k = j; k > i; k -= d)
				{
					if (A[k] < A[k - d])
						swap(A[k], A[k - d]);
				}
			}
		}
	}
}
```

### 快速排序

快速排序主要在于划分数组，将小于某个元素放在数组中左侧，大于的放在右侧。

```C++
int partition(vector<int> &A, int l, int r)
{
	int pivot = A[l];
	int m = l;
	for (int i = l + 1; i <=r; ++i)
	{
		if (A[i] < pivot)
		{
			swap(A[i], A[m + 1]);
			++m;
		}
	}
	swap(A[l], A[m]);
	return m;
}

void qucikSort(vector<int>& A,int l,int r)
{
	if (r <= l)
		return;
	int p = partition(A, l, r);
	qucikSort(A, l, p - 1);
	qucikSort(A, p+1, r);
}
```

平均情况下的时间复杂度为O(nlogn)

### 归并排序

将一个序列分成两个长度相等的子序列，为每一个子序列排序，然后再将它们合并成一个序列。合并两个子序列的过程称为归并。

```C++
void Merge(vector<int>& A, int left, int mid, int right)
{
	vector<int> B(right-left+1);
	int i = left, j = mid + 1;
	int k = 0;
	while (i <= mid && j <= right)
	{
		if (A[i] < A[j])
		{
			B[k++] = A[i++];
		}
		else
			B[k++] = A[j++];
	}

	while (i <= mid) {
		B[k++] = A[i++];
	}
	while (j <= right) {
		B[k++] = A[j++];
	}

	for (int i = left; i <= right; ++i)
	{
		A[i] = B[i - left];
	}
}

void MergeSort(vector<int>& A, int left, int right)
{
	if (left < right)
	{
		int mid = left + (right - left) / 2;
		MergeSort(A, left, mid);
		MergeSort(A,mid + 1, right);
		Merge(A, left, mid,right);
	}
}
```

logn层递归中，每一层都需要O(n)的时间代价，因此总的时间复杂度是O(nlogn)，需要一个和带排序数组大小相同的辅助数组，所以空间代价为O(n)

### 堆排序

构建最大堆，移出堆顶元素的顺序就是从大到小的顺序。首先是利用stl:

make_heap :在容器范围内，就地建堆，保证最大值在所给范围的最前面，其他值的位置不确定

pop_heap :堆顶(所给范围的最前面)元素移动到所给范围的最后，并且将新的最大值置于所给范围的最前面
push_heap :当已建堆的容器范围内有新的元素插入末尾后，应当调用push_heap将该元素插入堆中。

对应代码:

```C++
void heapsort(vector<int>&A)
{
	int mval;
	make_heap(A.begin(),A.end());
	for (int i = 0; i < A.size(); i++)
	{
		pop_heap(A.begin(),A.end()-i);
	}
}
```

如果需要建立最小堆，则可以结合lambda表达式来写cmp函数，结果就是从大到小的:

```C++
void heapsort(vector<int>&A)

}
```

也可以利用priority_queue进行操作:

如果不能用STL的话:

```C++
/********************************************
 * 向堆中插入元素
 *  hole：新元素所在的位置
 ********************************************/
template <class value>
void _push_heap(vector<value> &arr,int hole){
    value v = arr[hole];//取出新元素，从而产生一个空洞
    int parent = (hole - 1) / 2;
    //建最大堆，如果建最小堆换成 arr[parent] > value
    while(hole > 0 && arr[parent] < v){
        arr[hole] = arr[parent];
        hole = parent;
        parent = (hole - 1) / 2;
    }
    arr[hole] = v;
}

/********************************************
 * 删除堆顶元素
 ********************************************/
template <class value>
void _pop_heap(vector<value> &arr,int sz)
{
    value v = arr[sz - 1];
    arr[sz - 1] = arr[0];
    --sz;
    int hole = 0;
    int child = 2 * (hole + 1); //右孩子
    while(child < sz){
        if(arr[child] < arr[child - 1])
            --child;
        arr[hole] = arr[child];
        hole = child;
        child = 2 * (hole + 1);
    }
    if(child == sz){
        arr[hole] = arr[child - 1];
        hole = child - 1;
    }
    arr[hole] = v;
    _push_heap(arr,hole);
}

/********************************************
 * 建堆
 *  sz：删除堆顶元素后的大小
 *  v： 被堆顶元素占据的位置原来的元素的值
 ********************************************/
template <class value>
void _make_heap(vector<value> &arr)
{
    int sz = arr.size();
    int parent = (sz - 2) / 2;
    while(parent >= 0){
        int hole = parent;
        int child = 2 * (hole + 1); //右孩子
        value v = arr[hole];
        while(child < sz){
            if(arr[child] < arr[child - 1])
                --child;
            arr[hole] = arr[child];
            hole = child;
            child = 2 * (hole + 1);
        }
        if(child == sz){
            arr[hole] = arr[child - 1];
            hole = child - 1;
        }
        arr[hole] = v;
        _push_heap(arr,hole);
        --parent;
    }
}

template <class value>
void heap_sort(vector<value> &arr)
{
    _make_heap(arr);
    for(int sz = arr.size();sz > 1;sz--)
        _pop_heap(arr,sz);
}
```

## 单例模式

在开发或者设计中，可能会出现一个类只能有一个对象被创建的情况，如果有多个对象被创建，则可能会导致混乱。所以单例模式是恰当的解决办法。

先说说C++中的单例模式:

```C++
class singleton
{
protected:
    singleton(){}
private:
    static singleton* p;
public:
    static singleton* instance();
};
singleton* singleton::p = NULL;
singleton* singleton::instance()
{
    if (p == NULL)
        p = new singleton();
    return p;
}
```

构造函数声明为private或protected防止被外部函数实例化，内部保存一个private static的类指针保存唯一的实例，实例的动作由一个public的类方法代劳，该方法也返回单例类唯一的实例。 

但是如果是两个线程同时构造一个实例给p，两个线程就会同时检测到p为NULL然后生成实例，不唯一了，所以这个方法实现存在一定的问题。

### 懒汉与饿汉

懒汉模式:不到万不得已不会去实例化类

饿汉模式:在单例类定义的时候就实例化，值得注意的是饿汉模式下是线程安全的，因为在单例类定义的时候就已经定义了一个对象，对类进行了初始化。后面不管哪个线程调用成员函数initance()，都只不过是返回一个对象的指针而已。所以是线程安全的，不需要在成员函数initance中加锁 

下面实现懒汉模式:

```C++
class singleton
{
protected:
    singleton()
    {
        pthread_mutex_init(&mutex);
    }
private:
    static singleton* p;
public:
    static pthread_mutex_t mutex;
    static singleton* initance();
};

pthread_mutex_t singleton::mutex;
singleton* singleton::p = NULL;
singleton* singleton::initance()
{
    if (p == NULL)
    {
        pthread_mutex_lock(&mutex);  # 加锁
        if (p == NULL)
            p = new singleton();
        pthread_mutex_unlock(&mutex); # 解锁
    }
    return p;
}
```

或者可以用static实例，因为static定义可以保证拥有唯一实例，在返回时只要返回其指针就行了:

```C++
class singleton
```

## 哈希表简单实现

```C++
struct ListNode{
	int val;
	ListNode *next;
	ListNode(int x):val(x),next(NULL){}
};

int hash_func(int key, int table_len)
{
	return key % table_len;
}

void insert(ListNode *hash_table[], ListNode *node, int table_len)
{
	int hash_key = hash_func(node->val, table_len);
	node->next = hash_table[hash_key];
	hash_table[hash_key] = node;
}

bool search(ListNode *hash_table[], int value, int table_len)
{
	int hash_key = hash_func(value, table_len);
	ListNode *head = hash_table[hash_key];
	while (head)
	{
		if (head->val == value)
			return true;
		head = head->next;
	}
	return false;
}
```

5. 如果你对一个目录具有写权限，那么你是否具有对这个目录下的所有文件具有删除权限？

   要删除一个文件，你不一定要有这个文件的写权限，但你一定要有这个文件的上级目录的写权限。也就是说，你即使没有一个文件的写权限，但你有这个文件的上级目录的写权限，你也可以把这个文件给删除，而如果没有一个目录的写权限，也就不能在这个目录下创建文件。 

   但是，目录存在一个权限t(sticky bit)，设置该位后，即使用户对目录具有写权限，也不能删除该文件。

## Nginx

Nginx功能丰富，可作为HTTP服务器，也可作为反向代理服务器，邮件服务器。 

### Nginx常用功能

1.做Http代理和反向代理

Nginx在做反向代理时，提供性能稳定，并且能够提供配置灵活的转发功能。Nginx可以根据不同的正则匹配，采取不同的转发策略，比如图片文件结尾的走文件服务器，动态页面走web服务器，只要正则写的没问题，又有相对应的服务器解决方案，就可以得出正确的结果。并且Nginx对返回结果进行错误页跳转，异常判断等。如果被分发的服务器存在异常，他可以将请求重新转发给另外一台服务器，然后自动去除异常服务器。 

2.负载均衡

Ip hash算法，对客户端请求的ip进行hash操作，然后根据hash结果将同一个客户端ip的请求分发给同一台服务器进行处理，可以解决session不共享的问题。

上面是nginx的基本配置，需要注意的有以下几点：

1、`remote_addr `与`http_x_forwarded_for`用以记录客户端的ip地址； `remote_user`用来记录客户端用户名称； `time_local`用来记录访问时间与时区；`request`用来记录请求的url与http协议；`status`用来记录请求状态；成功是200， `body_bytes_s ent`记录发送给客户端文件主体内容大小；`http_referer`用来记录从那个页面链接访问过来的； `http_user_agent`记录客户端浏览器的相关信息；

2、惊群现象：一个网路连接到来，多个睡眠的进程被同时叫醒，但只有一个进程能获得链接，这样会影响系统性能。

3、每个指令必须有分号结束。

### 比Apache快

因为Apache使用的是select模型，而Nginx使用的是epoll模型。在高并发情况中，轮询I/O是最耗时间的操作之一。

##  正向代理与反向代理

正向代理相当于一个跳板机，代理访问外部资源。比如说我访问不了一个网站，但是我能访问一个代理服务器(能访问到呢个网站)，所以我可以连上代理服务器然后去访问呢个网站。

**正向代理 是一个位于客户端和原始服务器(origin server)之间的服务器，为了从原始服务器取得内容，客户端向代理发送一个请求并指定目标(原始服务器)，然后代理向原始服务器转交请求并将获得的内容返回给客户端。客户端必须要进行一些特别的设置才能使用正向代理。 **

反向代理实际运行方式是指以代理服务器来接受internet上的连接请求，然后将请求转发给内部网络上的服务器，并将从服务器上得到的结果返回给internet上请求连接的客户端，此时代理服务器对外就表现为一个服务器。 

正向反向实际上是看代理是请求外部网络还是内部网络来区分正反的。
