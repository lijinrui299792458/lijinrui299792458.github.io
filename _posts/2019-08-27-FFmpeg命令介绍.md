> FFmpeg命令介绍

在官网上下载的FFmpeg共有三个版本，其中Static是静态库版本、Shared则是动态库版本、Dev则是提供给开发者用的Lib文件。

## FFMPEG命令分类

### 基本信息查询命令

![命令类型](https://raw.githubusercontent.com/YunLambert/Blog_Pictures/master/img/20190825103009.png)

### 录制命令

```
ffmpeg -f avfoundation -i 1 -r 30 out.yuv
```

-f：指定使用avfoundation采集数据

-i：指定从哪儿采集数据，它是一个文件索引号

-r：指定帧率

文件索引号可以通过(mac上)进行查询

```
ffmpeg -f avfoundation -list_devices true -i ""
```

更多的devices信息可以在[https://www.ffmpeg.org/ffmpeg-devices.html#dshow](https://www.ffmpeg.org/ffmpeg-devices.html#dshow)上查询得到。

![list_devices](https://raw.githubusercontent.com/YunLambert/Blog_Pictures/master/img/20190825135711.png)

通过ffplay播放yuv文件时，由于yuv文件是原始文件，所以ffplay不知道yuv文件的分辨率，从而无法渲染，需要指定ffmpeg录制时产生的分辨率。同时由于播放的格式和ffmpeg生成的格式是不一样的，生成时的像素点是使用uyvy422格式的，所以ffplay播放时同样需要更改：

```
ffplay -s 2560*1600 -pix_fmt uyvy422 out.yuv
```



录音命令

```
ffmpeg -f avfoundation -i :0 out.wav
```



### 分解、复用命令 

多媒体格式转换

```
ffmpeg -i out.mp4 -vcodec copy -acodec copy out.flv
```

-i:输入文件

-vcodec copy:视频编码处理方式

-acodec copy:音频编码处理方式

eg.

```
ffmpeg -i test.mov -vcodec copy -acodec copy test.mp4 
```

直接提取出h264视频流

```
ffmpeg -i test.mov -an -vcodec copy out.h264
```

### 处理原始数据命令

提取YUV数据

```
ffmpeg -i input.mp4 -an -c:v rawvideo -pix_fmt yuv420p out.yuv
```

-c:v  用什么格式进行编码，rawvideo是使用视频原格式进行编码

提取PCM数据

```
ffmpeg -i out.mp4 -vn -ar 44100 -ac 2 -f s16le out.pcm
```

-ar:audio read 音频的采样率

-ac 2: audio channel 音频的声道，单、双、立体声

-f  s16le: 抽取出的PCM数据存储格式是，s是有符号的，16表示每一个数值用16位表示，le小端存储

播放时通过

```
ffplay -ar 44100 -ac 2 -f s16le out.pcm
```

### 裁剪、合并命令

音视频裁剪：

```
ffmpeg -i in.mp4 -ss 00:00:00 -t 10 out.ts
```

从什么时间点开始裁剪，裁剪10秒生成out.ts文件。

音视频合并：

```
ffmpeg -f concat -i inputs.txt out.flv
```

inputs.txt中是一个文件列表，记录了所有项合并的视频的名字。内容格式为：`file filename`

### 图片、视频互转命令

"视频实际上是一系列的图片的堆叠"，在识别视频时也是将视频提取成一张张图片，并对图片进行图像识别。

**视频转图片**

```
ffmpeg -i in.flv -r 1 -f image2 image-%3d.jpeg
```

-r:指定转换图片的帧率，1就是每秒钟转换1张图片

-f:将输入文件转换成什么格式的，image2是一种协议格式

**图片转视频**

```
ffmpeg -i image-%3d.jpeg out.mp4
```

### 直播相关命令

直播中最重要的就是推拉流

**直播推流**

```
ffmpeg -re -i out.mp4 -c copy -f flv rtmp://server/live/streamName
```

-re让帧率和源文件的帧率保持同步，-c是音视频编解码，-acodec音频编解码

**直播拉流**

```
ffmpeg -i rtmp://server/live/streamName -c copy dump.flv
```

拉取的是什么格式，保存下来就应该是什么格式的。

### 各种滤镜命令

“去水印”、“视频倍速”、“画中画”.....都可以通过滤镜来实现。

![ffmpeg处理音视频流程](https://raw.githubusercontent.com/YunLambert/Blog_Pictures/master/img/20190827151303.png)

我们需要对解码后的数据帧进行滤镜处理，然后重新进行编码最后完成ffmpeg滤镜。

```
ffmpeg -i in.mov -vf crop=in_w-200:in_h-200 -c:v libx264 -c:a copy out.mp4
```

-vf:视频滤镜，后面加上使用哪种滤镜，crop表示的是本身视频的宽度减少200，高度减少200.

-c:v 视频编码器使用的是libx264

-c:a 音频编码器使用的是copy，不进行处理

最后生成out.mp4文件。

crop格式：`crop=out_w:out_h:x:y`

ffmpeg加水印的一些技巧

**实现每隔10秒交替出现logo**

```
ffmpeg -y -t 60 -i input.mp4 -i logo1.png -i logo2.png -filter_complex "overlay=x=if(lt(mod(t\,20)\,10)\,10\,NAN ):y=10,overlay=x=if(gt(mod(t\,20)\,10)\,W-w-10\,NAN ) :y=10" output.mp4
```

`if(gt(mod(t,20),10),W-w-10,NAN )`中第二个10是logo出现的时长，即水印会显示10秒 

**实现2秒后logo从左到右移动**

```
ffmpeg -i input.mp4 -vf "movie=logo.png[logo];[in][logo]overlay=x='if(gte(t\,2)\,((t-2)*80)-w\,NAN)'" output.mp4
```

**实现gif格式的logo循环命令**

```
ffmpeg -i video.mpg -ignore_loop 0 -i loop.gif -filter_complex  "[0:v][1:v]overlay=10:10:shortest=1" -vcodec mpeg2video out.mpg
```

**实现视频添加字幕**

```
ffmpeg  -i  input  -vf  drawtext=fontfile=arial.ttf:text=welcome output
```

**添加水印**

```
ffmpeg –i input.flv -acodec copy-vcodec copy -vf "movie=test.png[watermark];[in][watermark]overlay=10:10:1[out]" output.flv
```

-i input.flv 表示要进行水印添加处理的视频
-acodec copy-vcodec copy表示保持音视频编码不变
-b 300k 表示处理视频的比特率
-vf “…” 中间便是水印处理参数，movie是指图片水印路径，搭配overlay一起使用，重要的是overlay=后面的部分，第一个参数表示水印距离视频左边的距离，第二个参数表示水印距离视频上边的距离，第三个参数 为1，表示支持透明水印。使用透明的png图片进行视频编码后，成功获得带透明水印的视频，并且画质也比较好。
