说一下ruby中的一个gem包:[whenever](https://github.com/javan/whenever)

## whenever的功能
可以实现后台进行定时读取信息或者执行脚本的周期性功能，官方文档中是这样介绍whenever的，它是一个编写和部署cron jobs的gem。什么是cron jobs，可以看一下[wiki](https://zh.wikipedia.org/wiki/Cron).

> crontab命令常见于Unix和类Unix的操作系统之中，用于设置周期性被执行的指令。该命令从标准输入设备读取指令，并将其存放于“crontab”文件中，以供之后读取和执行。该词来源于希腊语chronos（χρόνος），原意是时间。通常，crontab储存的指令被守护进程激活，crond常常在后台运行，每一分钟检查是否有预定的作业需要执行。这类作业一般称为cron jobs。

在这里说一下我的看法，crontab是用来安装、卸载或者列出定时任务列表的命令；然后在Unix操作系统中用于设置周期性执行指令的作业就是cron job。利用cron job，我们就可以配置某些命令或者脚本，让他们在某个设定的时间内周期性的运行。

而whenever事实上就是将ruby语言翻译成cron脚本，通过在ruby层控制代码，不需要对shell脚本进行处理，从而将周期性的任务交给cron来执行。

## 使用
首先安装:
```sh
gem install whenever
# Gemfile
gem 'whenever', require:false
```
然后命令行进入我们rails项目中，直接执行`wheneverize .`，就会自动创建一个初始文件:`config/schedule.rb`

看一下官方的例子:
```ruby
every 3.hours do # 1.minute 1.day 1.week 1.month 1.year is also supported
  runner "MyModel.some_process"
  rake "my:rake:task"
  command "/usr/bin/my_great_command"
end

every 1.day, at: '4:30 am' do
  runner "MyModel.task_to_run_at_four_thirty_in_the_morning"
end

every 1.day, at: ['4:30 am', '6:00 pm'] do
  runner "Mymodel.task_to_run_in_two_times_every_day"
end

every :hour do # Many shortcuts available: :hour, :day, :month, :year, :reboot
  runner "SomeModel.ladeeda"
end

every :sunday, at: '12pm' do # Use any day of the week or :weekend, :weekday
  runner "Task.do_something_great"
end

every '0 0 27-31 * *' do
  command "echo 'you can use raw cron syntax too'"
end

# run this task only on servers with the :app role in Capistrano
# see Capistrano roles section below
every :day, at: '12:20am', roles: [:app] do
  rake "app_server:task"
end
```
可以看到其中的every下出现了runner、command、rake，它是这样的:
```
job_type :command, ":task :output"
job_type :rake,    "cd :path && :environment_variable=:environment bundle exec rake :task --silent :output"
job_type :runner,  "cd :path && bin/rails runner -e :environment ':task' :output"
job_type :script,  "cd :path && :environment_variable=:environment bundle exec script/:task :output"
```
区别上面已经说的很清楚了，后面的`cd :path...`其实是Crontab指令，在这里也简单说一下:
```sh
crontab -l 显示目前排程 (list cron) 
crontab -e 编辑排程 (edit cron) 
crontab -r 移除排程 (remove cron) 
crontab -u 改变排程的执行身份（crontab -u user filename）
```
而我们之前看到的`cd :path && bin/rails runner -e :environment ':task' :output`，就是先移动到某个文件下然后到environment(production或者develop)执行:task任务。

总结一下就是command用的是最纯粹的shell command，rake用的是rake命令，runner用的是ruby的model，script则是某个脚本文件。

> 如果:path没有被设置，它将设定为default路径，即你运行whenever的路径。:environment_variable将被默认设定为RAILS_ENV下面，:environment将被默认设为producation。:output在默认请看下将被重定向到你的默认设置里。


## 一个例子
在这里让我们看一下怎么设置一个自动发邮件的程序。当然，让我们先看一下怎么发邮件:
### 发送邮件
**和python自动发送邮件是相同原理，了解程序自动发送邮件的可以忽略这一部分**
在Rails中发送电子邮件包含三个基本步骤:配置电子邮件的发送方式、确定何时发送电子邮价、以及提供电子邮件的内容。

首先应该在config/environments目录下的配置文件中确定邮件的发送方式:
```ruby
config.action_mailer.delivery_method= :stmp

config.action_mailer.smtp_settings = {
  address:  xx,
  port:     xx,
  domain:   xx,
  authentication: "plain",
  user_name: xx,
  password:  xx,
  enable_strattls_auto:true,
}
```
这个是基本设置，当然有些邮箱会设置第三方移动登陆的单独密码，这里的password是第三方的。
接着我们就可以通过`rails g mailer Order confirm_email`创建电子邮件的类名、电子邮件动作方法的名称。
```ruby
class OrderMailer < ApplicationMailer

  def confirm_email(order)
    @user = order.user
    @order = order
    mail(to: @user.email, subject: "订单 #{@order.number} 已经确认")
  end
end
```

具体的可以参照:[https://guides.rubyonrails.org/action_mailer_basics.html](https://guides.rubyonrails.org/action_mailer_basics.html).

如果像确保Email递送的可靠性，最好用第三方服务:
- mailgun
- SendGrid
- Postmark
- MailChimp
- AWS SES

### 具体步骤
首先在`lib/tasks`下看里一个daily_routine.rake，接着建立一个task:
```rake
desc "Send daily mail "
task :send_daily_mail do
  DailyMailer.send_notification.deliver_later!
end
```
在`app/mailer`下建立一个`daily_mailer.rb`的文件，接着再定义一个`send_notification`method可以通过这个方法发邮件。
```ruby
class OrderMailer < ApplicationMailer
  def  send_notification
    default to: -> { User.pluck(:email) },
    default :from "info@example.com"
    
    mail(
      subject: "just a test"
    )
  end
end
```
然后设置我们的`schedule.rb`文件，假设我们想每天早上10:00触发固定发送邮件:
```ruby
set :output, "log/cron_log.log"
env :PATH, ENV['PATH']
set :environment, :development
every 1.day, at: '10:00' do
  rake "send_daily_mail"
end
```
开始执行周期性任务:`whenever -i`.
在开发模式中如果实际寄送则很没效率并且不适合debug，所以可以在Gemfile中:`gem "letter_opener", :group => :development`，然后将`config.action_mailer.delivery_method`改成:`letter_opener`.这样就可以在开发者模式下，发送这个动作变成浏览器进行打开预览动作。
