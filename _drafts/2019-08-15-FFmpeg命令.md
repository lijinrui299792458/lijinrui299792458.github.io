### 录制命令

```
ffmpeg -f avfoundation -i 1 -r 30 out.yuv
```

-f：指定使用avfoundation采集数据

-i：指定从哪儿采集数据，它是一个文件索引号

-r：指定帧率

文件索引号可以通过(mac上)进行查询

```
ffmpeg -f avfoundation -list_devices true -i ""
```

更多的devices信息可以在[https://www.ffmpeg.org/ffmpeg-devices.html#dshow](https://www.ffmpeg.org/ffmpeg-devices.html#dshow)上查询得到。

![list_devices](https://raw.githubusercontent.com/YunLambert/Blog_Pictures/master/img/20190825135711.png)

通过ffplay播放yuv文件时，由于yuv文件是原始文件，所以ffplay不知道yuv文件的分辨率，从而无法渲染，需要指定ffmpeg录制时产生的分辨率。同时由于播放的格式和ffmpeg生成的格式是不一样的，生成时的像素点是使用uyvy422格式的，所以ffplay播放时同样需要更改：

```
ffplay -s 2560*1600 -pix_fmt uyvy422 out.yuv
```



录音命令

```
ffmpeg -f avfoundation -i :0 out.wav
```



### 分解、复用命令 

多媒体格式转换

```
ffmpeg -i out.mp4 -vcodec copy -acodec copy out.flv
```

-i:输入文件

-vcodec copy:视频编码处理方式

-acodec copy:音频编码处理方式

eg.

```
ffmpeg -i test.mov -vcodec copy -acodec copy test.mp4 
```

直接提取出h264视频流

```
ffmpeg -i test.mov -an -vcodec copy out.h264
```

### 处理原始数据命令

提取YUV数据

```
ffmpeg -i input.mp4 -an -c:v rawvideo -pix_fmt yuv420p out.yuv
```

-c:v  用什么格式进行编码，rawvideo是使用视频原格式进行编码

提取PCM数据

```
ffmpeg -i out.mp4 -vn -ar 44100 -ac 2 -f s16le out.pcm
```

-ar:audio read 音频的采样率

-ac 2: audio channel 音频的声道，单、双、立体声

-f  s16le: 抽取出的PCM数据存储格式是，s是有符号的，16表示每一个数值用16位表示，le小端存储

播放时通过

```
ffplay -ar 44100 -ac 2 -f s16le out.pcm
```
